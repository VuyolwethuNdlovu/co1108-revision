Deterministic Finite Automata

Brief Description of DFAs: 

An automaton is a theoretical machine which takes in an input and indicates whether this is accepted or not. DFA is good for accepting a language and allows you to compute a language. However, its simplicity does mean it is the least capable automata out of the various ones discovered. Deals with regular grammars. 

It has several components: 

  • Alphabet ( Collection of symbols to construct strings in your language) 
  • Set of states (Records your steps and represents the memory of your machine) 
  • Starting state (Initial state â€“ Machine boots up) 
  • Transitions (partial) function
  • Set of final (accepting) states (Machine terminates, string is accepted) 

Key point from the tutorial: 

Problem with DFA is that you CANNOT have multiple transitions from a single state.
