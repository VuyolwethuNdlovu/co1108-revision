 Nondeterministic Finite Automata

Brief Description of NFAs: 

An NFA has increased functionality compared to DFA. It
allows more than one transition (choice) from a state and can include an empty
string transition. Both DFA and NFA are similar but NFA’s are more concise and
allow you to use fewer states when solving a language.





Key point from the tutorial: 

Use {} to refer to multiple states (E.g. {s1,s2})

1. From state s0 see what states the transition ‘a’ leads to. This creates the state {s1, s3} in the
solution.

2. Trace which states ‘a’ transitions are made to. Hence the state {s3} is created.

3. Trace which states ‘b’ transitions are made to. Hence the self-loop to {s1, s3} in the solution.

4. Trace which states ‘c’ transitions are made to. Hence the transition to s2 from {s1, s3}.

5. S1 has no transitions of ‘a’ however s3 does. Hence the self-loop to s3.

6. From s3 only a ‘c’ transition exists to s2. Hence the transition from s3 to s2.
